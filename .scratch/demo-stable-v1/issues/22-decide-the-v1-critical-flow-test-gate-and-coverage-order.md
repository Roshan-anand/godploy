# Decide the V1 critical-flow test gate and coverage order

Status: hard-human

## What to build

Turn the PRD's testing direction into a concrete V1 decision. The output should define which critical flows must be covered before demo-stable release, which test layer owns each flow, and what order the team should implement that coverage in.

## Acceptance criteria

- [ ] The critical V1 flows are listed explicitly rather than left as an implied future follow-up.
- [ ] Each critical flow is assigned an intended test layer or verification style.
- [ ] The result is recorded as a concrete backlog or execution order that AFK implementation work can follow.

## Blocked by

None - can start immediately
